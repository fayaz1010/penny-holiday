Vercel + GitHub (no secrets in this repo)
=========================================

0) If `git push` rejects workflow files: your GitHub token needs the `workflow` scope.
   Run:  gh auth refresh -h github.com -s workflow
   Complete the browser/device flow, then push again.

1) GitHub repository secrets — automated from vault (run locally after `gh auth login`):
      npm run sync-vercel-secrets
   Or set manually (Settings → Secrets and variables → Actions):
   - VERCEL_TOKEN      → vault "vercel.token"
   - VERCEL_ORG_ID     → Vercel team/account id
   - VERCEL_PROJECT_ID → Vercel project id (script creates project "automore" if missing)

2) Vercel project: import the GitHub repo, framework Next.js, root directory "/",
   set Production env: NEXT_PUBLIC_SITE_URL=https://automore.com.au

3) First-time link (optional, to generate .vercel locally — do not commit .vercel/):
   npx vercel link

4) Push to `main` runs `.github/workflows/vercel-production.yml` and deploys.

Security: if vault JSON was ever exposed in a chat or log, rotate GitHub + Vercel tokens in GitHub/Vercel settings.
